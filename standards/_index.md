# Standards Index

Each file contains the essential and desirable attributes for one
ACM SIGSOFT Empirical Standard.
Source: https://acmsigsoft.github.io/EmpiricalStandards/ — CC0 license.

## Available Standards

| File | Method | Apply when... |
|------|--------|---------------|
| general.md | General | ALWAYS — applies to every empirical paper |
| action-research.md | Action Research | Researcher intervenes in a real organization |
| benchmarking.md | Benchmarking | Software system assessed using a benchmark |
| case-study.md | Case Study / Ethnography | Deep investigation of one phenomenon at one site |
| case-survey.md | Case Survey | Systematic conversion of qualitative cases to quantitative data |
| data-science.md | Data Science | Analysis of SE phenomena using ML or computational methods |
| experiment.md | Experiment | Controlled study with human participants |
| grounded-theory.md | Grounded Theory | Iterative data collection and analysis toward theory |
| longitudinal.md | Longitudinal Study | Repeated observations of same variables over time |
| meta-science.md | Meta-Science | Methodological guidelines or analysis of research methodology |
| mixed-methods.md | Mixed Methods | Two or more methods combined (also apply each method's standard) |
| optimization.md | Optimization Study | Search-based or optimization algorithm evaluation |
| qualitative-survey.md | Qualitative Survey | Broad qualitative data collection without iterating |
| quantitative-simulation.md | Quantitative Simulation | Simulation of models of software systems |
| questionnaire-survey.md | Questionnaire Survey | Structured quantitative survey instrument |
| replication.md | Replication | Deliberate repetition of a prior study |
| repository-mining.md | Repository Mining | Analysis of data from software repositories |
| engineering-research.md | Engineering Research | Building and evaluating a software tool, framework, or artifact |
| systematic-review.md | Systematic Review | Systematic search and synthesis of prior literature |

## Tier coverage

These standards include **Essential** and **Desirable** attributes from the
ACM SIGSOFT Empirical Standards. The upstream standards also define an
**Extraordinary** tier for exceptional contributions. That tier is omitted here
because `/draft-review` focuses on pass/fail evaluation of methodological soundness
rather than ranking excellence. If you need the full Extraordinary criteria,
consult the [interactive checklists](https://acmsigsoft.github.io/EmpiricalStandards/).

## For fields outside software engineering

No validated standard exists beyond general.md.
/draft-review will apply general.md only and mark the review PROVISIONAL.
The researcher should treat a PROVISIONAL review as directional, not authoritative.

A domain expert must co-sign any community-contributed standard before it
enters the registry. AI-drafted standards for other fields are marked
`experimental` until co-signed.
